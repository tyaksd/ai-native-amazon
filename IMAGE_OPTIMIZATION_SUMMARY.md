# 画像最適化コスト削減の実装概要

Vercelの画像最適化コストを最小限に抑えるための最適化を実装しました。

## 実装した最適化

### 1. Next.js設定の最適化 (`next.config.ts`)
- **キャッシュ期間の延長**: `minimumCacheTTL: 2678400` (31日) を設定
- **画像フォーマットの制限**: WebPのみに制限して変換数を削減
- **品質の最適化**: デフォルト品質を80に設定
- **デバイスサイズの制限**: 不要なサイズの変換を削減
- **画像サイズの制限**: 必要最小限のサイズのみをサポート

### 2. カスタム画像ローダーの最適化 (`lib/imageLoader.ts`)
- **WebPフォーマット固定**: `f_webp`で変換数を削減
- **品質の最適化**: デフォルト品質を80に設定
- **プログレッシブJPEG**: 読み込み速度の向上

### 3. 画像最適化ユーティリティ (`lib/imageOptimization.ts`)
- **最適化回避の判定**: SVG、GIF、小さな画像（16x16以下）は最適化を回避
- **品質の動的最適化**: 画像サイズに応じて品質を調整
- **優先度の最適化**: 重要な画像のみ高優先度に設定

### 4. 最適化されたImageコンポーネント (`app/components/OptimizedImage.tsx`)
- **自動最適化判定**: 画像の種類とサイズに応じて自動的に最適化を制御
- **コスト削減**: 不要な変換を避けてコストを削減
- **パフォーマンス向上**: 適切な品質と優先度でパフォーマンスを向上

### 5. 既存コンポーネントの最適化
- **メインページ**: 商品画像とブランド画像を最適化
- **ProductCarousel**: カルーセル内の画像を最適化
- **重要度の設定**: 重要な画像には`isImportant={true}`を設定

## 期待される効果

1. **変換数の削減**: WebPフォーマット固定により変換数を削減
2. **キャッシュ効率の向上**: 31日のキャッシュ期間で再変換を削減
3. **不要な変換の回避**: 小さな画像やSVG、GIFは最適化を回避
4. **品質の最適化**: 画像サイズに応じた適切な品質設定
5. **コストの削減**: 全体的な画像最適化コストの大幅な削減

## 使用方法

既存の`Image`コンポーネントを`OptimizedImage`に置き換えるだけで、自動的に最適化が適用されます。

```tsx
// 従来
<Image src={src} alt={alt} width={800} height={800} />

// 最適化後
<OptimizedImage src={src} alt={alt} width={800} height={800} isImportant={true} />
```

## 注意事項

- 小さな画像（16x16以下）は自動的に`unoptimized`になります
- SVGとGIFファイルは自動的に`unoptimized`になります
- 重要な画像には`isImportant={true}`を設定してください
